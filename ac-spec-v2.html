Auto-paste rule
When a session is active and the user copies anything to clipboard on their Mac, it is automatically pasted into the AI canvas. This works across all 4 modes. The intent is zero-friction context feeding — copy a Slack message, an email, a code snippet — it flows in without switching to the widget.

Transcript mode removes filler words, corrects grammar, and makes your raw speech readable. It then optionally converts the cleaned text into a structured AI prompt — useful when you want to ask another AI a well-formed question.

Capability & skill matching
HOW AI PICKS A SKILL
Llama analysis extracts task intent and tool requirements. skills.db is queried for matching skill by intent keywords. If a skill is found, it's attached to the task. If not, task is flagged no_skill and user is told to create or connect the capability.
NO SKILL STATE
Task is created, stored, visible in panel. A warning shown: "No capability found for this task. Set up a skill in Settings → Skills to execute it." User can set up the skill and press Rerun on the task log entry.
PROJECT ROUTING
If tasks mention a project name that matches an existing project in projects.db, they're auto-routed. If not, the project selector is shown on session end. Tasks without projects go to a default "Inbox" project.

Knowledge sources
🧠 World model
Persistent facts, decisions, people, and entities extracted from all sessions. Updated after each run.
📂 Session history
All previous transcripts, cleaning passes, and analysis outputs. Queryable by date, project, person.
📋 Project context
Tasks, completions, and decisions per project. When a project is mentioned in the question, its context is prioritised.
🌐 AI knowledge
General world knowledge from the LLM. Always available as fallback when personal context doesn't have the answer.
ATTRIBUTION
Answer always shows source attribution at bottom: which sessions, world model, or extractions were used. Keeps the answer honest and traceable.

How Meeting mode works
Meeting mode is designed for zero-friction note-taking. You don't have to tag speakers, bullet things down, or manage formatting. Just start a session, let it run, end it.
What happens during the session
WhatHow
Speaker differentiationTone analysis and voice print clustering. Labelled Speaker A, B, C — not by name (unless mentioned by others).
Live transcript scrollCanvas shows rolling transcript. Each speaker's lines shown in different opacity to differentiate.
Analysis + ExecutionOFF by default during meeting. Only MIC is active. Processing happens at end to avoid latency during conversation.

SPEAKER NAMING
If someone in the meeting says a name (e.g. "Vivek, can you do this?"), that label is matched against the People database. If a match exists, the speaker is identified by name from that point in the transcript.
